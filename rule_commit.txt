Type	Ý nghĩa (Dùng khi nào?)
feat	(Feature) Khi bạn thêm một chức năng mới hoàn toàn.
fix	Khi bạn sửa một lỗi (bug).
docs	Khi bạn chỉ thay đổi tài liệu, ghi chú (như file main.txt của bạn).
style	Thay đổi định dạng code (khoảng trắng, dấu phẩy...) không làm thay đổi logic.
refactor	Sửa lại code cho sạch hơn, tối ưu hơn nhưng chức năng vẫn giữ nguyên.
test	Khi bạn thêm hoặc sửa các bài kiểm tra (unit test).
chore	Các thay đổi nhỏ nhặt về cấu hình, cập nhật thư viện...

QUY TẮC COMMIT CODE
1. Commit là phải chạy được

Chỉ commit khi code không lỗi.

Không commit trong trạng thái “đỏ lòm”.

Nếu buộc phải dừng giữa chừng → commit và ghi rõ: WIP (Work In Progress).

2. Một thay đổi – Một commit

Mỗi commit chỉ nên chứa một mục đích duy nhất.

Không gộp nhiều thay đổi không liên quan vào cùng một commit.

Hoàn thành một việc nhỏ → commit ngay.

3. Xong đâu – Commit đó

Đừng đợi cuối buổi mới commit.

Hoàn thành một phần nội dung → lưu lại ngay.

Commit sớm giúp dễ dàng quay lại phiên bản trước khi cần.

4. Viết message cho “mình của 6 tháng sau”

Message phải trả lời được: Làm gì? Ở đâu?

Tránh các message chung chung như: update, fix, edit.

Hãy viết đủ rõ để sau này đọc lại vẫn hiểu mình đã thay đổi điều gì.
